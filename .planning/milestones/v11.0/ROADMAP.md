- Add `--speculative-depth N` flag to control how many phases ahead to plan (default: 2)

### Phase 22: Autonomous Error Recovery
**Goal:** Add graduated error recovery to autonomous mode — retry, diagnose, and continue instead of hard-stopping on every failure
**Requirements:**

### Phase 24: Incident Journal System
**Goal:** Add a structured, append-only incident journal that automatically captures every issue PBR encounters during any workflow — for later review, audit queries, and pattern learning
**Requirements:**
- Create `.planning/incidents/` directory with individual JSONL files per day (`incidents-YYYY-MM-DD.jsonl`)
- Structured entry format: `{ timestamp, session_id, source (hook/skill/agent), type (block/warn/error/retry/deviation/contention), severity (info/warning/error), issue (description), context (phase/plan/task/file), auto_fixed (boolean), resolution (description or null), duration_ms }`
- Add `pbr-tools.cjs incidents record|list|query` CLI commands for manual recording and querying
- Hook integration (non-blocking): PreToolUse hooks that block → auto-record with `type: "block"`, PostToolUse hooks that warn → auto-record with `type: "warn"`, `log-tool-failure.js` failures → auto-record with `type: "error"`

- Audit integration: add incident query as a dimension in the audit system so `/pbr:audit` can surface incident patterns
- Config toggle: `features.incident_journal: true` (default true) — can be disabled if unwanted
- NEVER block workflow — all recording is fire-and-forget with try/catch. If the write fails, silently skip.
**Success criteria:** Every hook block, warning, agent deviation, retry, and multi-session contention event is automatically captured. `incidents list` shows recent entries. `incidents query` filters work. Audit can query incidents. Zero performance impact on normal workflows.
