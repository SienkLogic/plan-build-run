# Plan 14-03: Create Research Project Output Templates

Create `templates/research/` subdirectory with 5 research output templates adapted from GSD's `templates/research-project/`. These templates define the output format for the towline-researcher agent when producing research project documents.

<task id="14-03-T1" type="auto" tdd="false">
  <name>Create templates/research/ directory with ARCHITECTURE, FEATURES, and PITFALLS templates</name>
  <files>
    plugins/dev/templates/research/ARCHITECTURE.md.tmpl
    plugins/dev/templates/research/FEATURES.md.tmpl
    plugins/dev/templates/research/PITFALLS.md.tmpl
  </files>
  <action>
1. Create directory `plugins/dev/templates/research/`

2. Create `plugins/dev/templates/research/ARCHITECTURE.md.tmpl`:
   - Adapt from GSD `templates/research-project/ARCHITECTURE.md` (204 lines)
   - Add header: `<!-- Source: templates/research/ARCHITECTURE.md.tmpl | Purpose: Research output format for technology architecture analysis -->`
   - Use `{variable}` placeholder syntax for dynamic content (project name, technology, etc.)
   - Replace GSD-specific references:
     - `/gsd:*` commands -> `/dev:*` commands
     - GSD agent names -> Towline agent names
     - GSD directory paths -> Towline paths (`.planning/phases/`, etc.)
   - Template should define sections for: architectural overview, component diagram, data flow, deployment model, scalability considerations
   - Use UPPER.md.tmpl filename to match Towline convention (GSD uses lowercase)

3. Create `plugins/dev/templates/research/FEATURES.md.tmpl`:
   - Adapt from GSD `templates/research-project/FEATURES.md` (147 lines)
   - Add header: `<!-- Source: templates/research/FEATURES.md.tmpl | Purpose: Research output format for technology feature analysis -->`
   - Template should define sections for: feature inventory, API surface, configuration options, extension points, limitations
   - Same placeholder and naming conventions as ARCHITECTURE.md.tmpl

4. Create `plugins/dev/templates/research/PITFALLS.md.tmpl`:
   - Adapt from GSD `templates/research-project/PITFALLS.md` (200 lines)
   - Add header: `<!-- Source: templates/research/PITFALLS.md.tmpl | Purpose: Research output format for technology pitfalls and risk analysis -->`
   - Template should define sections for: common mistakes, performance pitfalls, security concerns, migration risks, version compatibility issues
   - Same placeholder and naming conventions
  </action>
